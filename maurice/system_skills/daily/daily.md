Daily owns the morning synthesis surface.

Build the digest from the current agent's latest dream report and from the
`daily.md` contributions declared by loaded skills. Treat those contributions as
instructions about what belongs in the daily, not as direct user-facing text.

The digest can mention other agents only when a loaded skill explicitly
contributes multi-agent context. Agent memory, notes, todos, dreams, and
reminders remain agent-scoped by default.

Prefer concise sections:

- active projects or topics that need attention
- useful ideas discovered by dreaming
- proposed actions that still need user approval
- links between projects or agents when a skill provides them
